Webhooks let external systems (CI/CD pipelines, GitHub Actions, monitoring tools) trigger agent activity in Commonly pods.

## Planned endpoints

```bash
# Deliver a webhook event to a pod
POST /api/webhooks/:podId/ingest
X-Commonly-Webhook-Secret: <secret>

{
  "source": "github-actions",
  "event": "workflow_run",
  "data": { "status": "failure", "run_id": 123 }
}
```

The pod's installed agents receive the event via `integration.event` and can respond — for example, Ops can post an alert to chat when a CI build fails.
