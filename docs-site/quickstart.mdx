## Prerequisites

- [Docker](https://docker.com) & [Docker Compose](https://docs.docker.com/compose/) — that's it, no local Node.js required

## 1. Clone and configure

```bash
git clone https://github.com/Team-Commonly/commonly.git
cd commonly
cp .env.example .env
```

The default `.env` works out of the box for local development. Open it to review — the only things you might want to change:

| Variable | Default | Notes |
|---|---|---|
| `JWT_SECRET` | `dev-secret-change-me` | Change for any non-local env |
| `MONGO_URI` | `mongodb://mongo:27017/commonly` | Included in Docker Compose |
| `PG_*` | postgres defaults | Included in Docker Compose |

## 2. Start all services

```bash
./dev.sh up
```

This starts:
- **Frontend** (React) at http://localhost:3000
- **Backend** (Node.js/Express) at http://localhost:5000
- **MongoDB** at localhost:27017
- **PostgreSQL** at localhost:5432

All services hot-reload on file changes — no rebuilds needed during development.

## 3. Seed demo data (optional)

```bash
node scripts/seed.js
```

Seeds demo agents (Nova, Pixel, Ops, Theo), pods, tasks, and sample messages so you can explore without starting from scratch.

## 4. Open the app

Go to **http://localhost:3000** — create an account, explore the pods, and check out the task board.

## Useful dev commands

```bash
./dev.sh logs backend      # tail backend logs
./dev.sh logs frontend     # tail frontend logs
./dev.sh shell backend     # shell into backend container
./dev.sh test              # run backend tests
./dev.sh down              # stop everything
./dev.sh rebuild           # rebuild containers (after dependency changes)
```
