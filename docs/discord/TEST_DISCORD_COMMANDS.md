# Testing Discord Slash Commands

This guide covers how to test and verify Discord slash command functionality in the Commonly platform.

## Prerequisites

- Active Discord integration set up in Commonly
- Discord bot with proper permissions
- Access to the Discord server where the bot is installed

## Test Environment Setup

### 1. Verify Integration Status

First, ensure your Discord integration is properly configured:

```bash
# Check integration status
curl -X GET http://localhost:3000/api/discord/binding/POD_ID
```

Expected response:
```json
{
  "integration": {
    "id": "integration_id",
    "type": "discord",
    "status": "connected",
    "config": {
      "serverId": "guild_id",
      "serverName": "Test Server",
      "channelId": "channel_id",
      "channelName": "test-channel"
    }
  }
}
```

### 2. Register Slash Commands

Ensure slash commands are registered for your Discord server:

```bash
# Register commands for a specific integration
curl -X POST http://localhost:3000/api/discord/register-commands/INTEGRATION_ID
```

Expected response:
```json
{
  "message": "Slash commands registered successfully",
  "commands": [
    "/commonly-summary",
    "/discord-status",
    "/discord-enable",
    "/discord-disable"
  ]
}
```

## Manual Testing

### Test 1: `/commonly-summary` Command

**Steps:**
1. Open Discord and navigate to your server
2. Type `/commonly-summary` in any channel
3. Press Enter to execute the command

**Expected Results:**
- If summaries exist: Returns formatted summary with time period, message count, and content
- If no summaries: Returns "No recent summaries available for this chat pod"
- If integration not found: Returns error message (ephemeral)

**Example Response:**
```
📊 Chat Pod Summary

⏰ Time Period: 12/1/2023, 2:00:00 PM - 12/1/2023, 3:00:00 PM
💬 Messages Analyzed: 15
📝 Summary Type: hourly

The team discussed the new feature implementation and decided on the technical approach...

---
*Generated by Commonly AI*
```

### Test 2: `/discord-status` Command

**Steps:**
1. Type `/discord-status` in Discord
2. Press Enter to execute

**Expected Results:**
- Shows integration status with emojis
- Displays server name, channel name, webhook status
- Shows last sync timestamp

**Example Response:**
```
🤖 Discord Integration Status

📊 Status: 🟢 connected
🏠 Server: Test Server
📺 Channel: #general
🔗 Webhook Listener: ✅ Enabled
⏰ Last Sync: 12/1/2023, 3:30:45 PM
```

### Test 3: `/discord-enable` Command

**Steps:**
1. Type `/discord-enable` in Discord
2. Press Enter to execute

**Expected Results:**
- Confirms webhook listener is enabled
- Updates integration configuration
- Shows success message

**Example Response:**
```
✅ Webhook listener enabled! Discord channel activity will now be summarized and sent to the chat pod.
```

### Test 4: `/discord-disable` Command

**Steps:**
1. Type `/discord-disable` in Discord
2. Press Enter to execute

**Expected Results:**
- Confirms webhook listener is disabled
- Updates integration configuration
- Shows success message

**Example Response:**
```
❌ Webhook listener disabled. Discord channel activity will no longer be summarized.
```

### Unit Testing

Create unit tests for individual command functions:

```javascript
// Example test for status command
describe('DiscordCommandService', () => {
  it('should handle status command correctly', async () => {
    const commandService = new DiscordCommandService('test_installation_id');
    const result = await commandService.handleStatusCommand();
    
    expect(result.success).toBe(true);
    expect(result.content).toContain('Discord Integration Status');
  });
});
```

## Error Scenarios

### Test Error Handling

1. **Invalid Installation ID**
   - Use a non-existent installation ID
   - Expected: Error message about integration not found

2. **Missing Permissions**
   - Remove bot permissions in Discord
   - Expected: Permission error messages

3. **Network Issues**
   - Disconnect from internet
   - Expected: Connection timeout errors

4. **Invalid Command**
   - Type `/invalid-command`
   - Expected: "Unknown command" error

## Performance Testing

### Load Testing

Test command response times under load:

```bash
# Test multiple concurrent commands
for i in {1..10}; do
  curl -X POST http://localhost:3000/api/discord/interactions \
    -H "Content-Type: application/json" \
    -d '{"type":2,"data":{"name":"discord-status"}}' &
done
wait
```

**Expected Results:**
- Response time < 2 seconds for each command
- No timeout errors
- Consistent response format

### Rate Limiting

Test Discord API rate limits:

```bash
# Send multiple commands rapidly
for i in {1..50}; do
  # Simulate command execution
  sleep 0.1
done
```

**Expected Results:**
- Commands processed successfully
- No rate limit errors
- Proper error handling if limits exceeded

## Debugging

### Common Issues

1. **Commands Not Appearing**
   - Check bot permissions (`applications.commands` scope)
   - Verify command registration was successful
   - Check Discord API status

2. **Permission Errors**
   - Ensure bot has `Send Messages` permission
   - Check channel-specific permissions
   - Verify bot role hierarchy

3. **Response Delays**
   - Check network connectivity
   - Monitor Discord API response times
   - Review server logs for bottlenecks

### Log Analysis

Monitor server logs during testing:

```bash
# Watch server logs
tail -f logs/discord.log | grep "slash command"

# Check for errors
grep "ERROR" logs/discord.log | grep "discord"
```
